# OpenPet v1.0.1-rc.3

OpenPet v1.0.1-rc.3 is a release-candidate update for the desktop pet platform. It keeps the project macOS-first for packaged validation while strengthening the Control Center, AI setup, Creator Studio, plugin runtime, cursor behavior, and release evidence path.

## Highlights

- Creator Studio can now carry more of the pet/action creation loop: prompt planning, generated atlas handling, frame repair, approval, dashboard review, and host-mediated import flows.
- AI setup is clearer and safer, with improved provider save/test behavior, memory/persona work, image-provider guidance, and main-process secret handling.
- Plugin runtime work is broader and cleaner: declaration command/service/dashboard controls, bridge routes, loopback health checks, logs, storage stats, cleanup evidence, and a refactored plugin service support layer.
- Desktop polish improved across cursor rendering, hotspots, context menus, walking, viewport scaling, settings window stacking, and narrow Control Center layouts.

## Release Notes

- This is still a release candidate. macOS is the primary validated desktop target.
- Windows packaging and smoke evidence tooling are present, but Windows is not being advertised as release-ready until signed installer evidence and real smoke reports are archived.
- GitHub Actions release builds are expected to attach platform artifacts to this release after the tag workflow finishes.
